Objetivos do Projeto

Objetivo Principal

Fornecer insights e dados qualificados sobre as relações de citação direta em decisões de cortes superiores em casos de litigância climática, especialmente os fundados no Acordo de Paris e baseados em pareceres, relatórios e pesquisas/artigos do IPCC, considerando cortes citando decisões de outras cortes em suas próprias decisões.

Objetivo Específico: Norte Global vs. Sul Global

Identificar o número relativo ou proporção de citações feitas inter cortes do Norte Global, referenciando-se; versus o número relativo ou proporção de decisões do Sul Global referenciando as cortes do Norte; versus citações de cortes do Sul Global feitas por cortes do Norte.

Contexto Acadêmico: Este projeto integra pesquisa de doutorado sobre litigância climática, com rigor metodológico e documentação completa de todos os processos executados para alcançar as informações de interesse hábeis a suportar a tese acadêmica.

Escopo dos Dados

Formato

CSV estruturado contendo metadados e referências para documentos externos

Dimensão

Aproximadamente 15.000 casos de múltiplas jurisdições

Idiomas

Múltiplos idiomas (cortes internacionais em inglês/francês; cortes nacionais em idiomas oficiais)

Documentos

Decisões finais em PDF (maioria texto-nativo; ~20% podem requerer OCR - será ignorado)

Definição Operacional: Norte vs. Sul Global

Norte Global: EUA, Canadá, Europa Ocidental, Austrália e Nova Zelândia
Sul Global: Demais jurisdições

Nota: Esta taxonomia será refinada e documentada metodologicamente na Fase 0.

Roadmap: Estrutura em Fases

FASE 0: Fundação e Preparação

Duração: 3-4 dias | Período: 16-20 outubro

Objetivo

Estabelecer infraestrutura, entender os dados, definir metodologia

Entregas

  • Schema completo do CSV (colunas, tipos de dados, padrões)
  • Taxonomia operacional de jurisdições (Norte vs Sul Global)
  • Ambiente de desenvolvimento configurado (Python + bibliotecas necessárias)
  • Estratégia de sampling para testes (ex: 100 casos para validação antes de processar 15k)
  • Definição de padrões de citação a serem identificados
  • Documento metodológico inicial (para a tese)

Ferramentas

pandas, jupyter notebooks, Git para versionamento

FASE 1: Extração e Preprocessing

Duração: 7-10 dias | Período: 21 outubro - 31 outubro

Objetivo

Obter o texto das decisões de forma estruturada e processável

Sub-etapas

1.1 - Mapeamento e Download
  • Script para baixar PDFs das decisões finais
  • Handling de erros (links quebrados, timeouts, etc.)
  • Organização em estrutura de diretórios lógica
1.2 - Extração de Texto
  • PDFs texto-nativos: PyPDF2 ou pdfplumber
  • PDFs escaneados: Ignorados (representam <20% da base)
  • Validação de qualidade da extração
1.3 - Estruturação e Metadados
  • Criar banco de dados estruturado (SQLite local ou PostgreSQL)
  • Schema: caso_id, jurisdição, corte, ano, região (Norte/Sul), texto_decisão, idioma, path_original
  • Preprocessamento linguístico básico (sem tradução ainda)

Nota sobre OCR: Aproximadamente 20% dos documentos podem ser PDFs escaneados. Para cumprir o prazo de 30 de novembro, estes serão ignorados. Uma base de 80% (~12.000 casos) é estatisticamente robusta para identificar tendências e padrões de citação.

FASE 2: Identificação de Citações

Duração: 10-12 dias | Período: 1 novembro - 13 novembro

Objetivo

Extrair todas as citações explícitas inter-cortes

Abordagem Híbrida

2.1 - Regex e Padrões (gratuito, rápido, limitado)
  • Identificar padrões comuns de citação judicial
    • Ex: "Smith v. Jones (2019)", "Case C-123/45", "RE 123456"
    • Nomes de cortes seguidos de datas/números
    • Variações por jurisdição
  • Criar biblioteca de padrões por tipo de corte
2.2 - NER (Named Entity Recognition) com modelos open-source
  • spaCy com modelo jurídico (legal-bert ou equivalente)
  • Identificar entidades tipo "CASE", "COURT", "DATE"
  • Custo: zero, mas precisão moderada
2.3 - LLM-powered extraction (custo controlado, alta precisão)
  • Usar Claude/GPT apenas para casos onde regex+NER falharam ou são ambíguos
  • Prompt estruturado pedindo JSON com citações
  • Estratégia de custo:
    • Processar apenas seções relevantes (não o PDF inteiro)
    • Identificar seções com alta probabilidade de conter citações (ex: "jurisprudência citada", "precedentes", "fundamentação")
    • Batch processing para eficiência
2.4 - Validação e Matching
  • Cross-reference: citações identificadas correspondem a casos existentes na base?
  • Resolver ambiguidades (casos com nomes similares)
  • Criar matriz de citações: caso_citante → caso_citado

Output desta fase

  • Tabela normalizada de citações
  • Grafo direcionado de citações (NetworkX)

FASE 3: Análise Quantitativa

Duração: 5-7 dias | Período: 14 novembro - 21 novembro

Objetivo

Responder às perguntas de pesquisa sobre fluxos de citação Norte-Sul Global

Análises Principais

3.1 - Métricas Descritivas
  • Total de citações por região (Norte→Norte, Norte→Sul, Sul→Norte, Sul→Sul)
  • Proporções relativas e absolutas
  • Distribuição temporal (evolução ao longo dos anos)
  • Cortes mais citadas (ranking)
  • Cortes mais "citadoras" (ranking)
3.2 - Análise de Rede
  • Centralidade (betweenness, eigenvector) - quais cortes são "hubs"?
  • Comunidades (existem clusters regionais além de Norte/Sul?)
  • Caminhos: uma decisão do Sul que cita o Norte tende a ser citada por quem?
3.3 - Testes Estatísticos
  • A proporção Norte→Sul é significativamente diferente de Sul→Norte?
  • Qui-quadrado, testes de proporção
  • Regressões se houver variáveis temporais/contextuais

Ferramentas

pandas, NetworkX, scipy.stats, statsmodels

Escalabilidade para Fase 2 (Correntes Doutrinárias)

A arquitetura proposta já contempla escalabilidade para análises futuras mais complexas:

Embedding Semântico

Em vez de só identificar citações explícitas, podemos gerar embeddings dos argumentos de cada decisão usando modelos multilíngue como multilingual-e5 ou via API

Clustering Temático

Agrupar decisões por similaridade argumentativa

Topic Modeling

LDA ou BERTopic para identificar temas recorrentes

Análise de Influência Implícita

Mesmo sem citação direta, decisões podem usar argumentos similares

Reutilização de Infraestrutura: A infraestrutura de dados e pipeline de processamento criados na fase atual serão completamente reutilizáveis para análises de segunda fase.

Estimativa de Custos de API

Cenário Conservador

  • 15k decisões × média 30 páginas = 450k páginas
  • Assumindo que precisamos de LLM para ~20% dos casos (onde regex falha)
  • 3k decisões × ~15k tokens cada = 45M tokens de input
  • Com Claude Sonnet: ~$3 per 1M input tokens = ~$135
  • Output (citações em JSON): ~5M tokens = ~$15

Total estimado: $150-200 para extração de citações

Estratégias de Redução de Custos

  • Processando apenas seções relevantes (não decisão inteira)
  • Usando modelos menores para casos simples
  • Batching inteligente
  • Cache de resultados para evitar reprocessamento

Limitações Metodológicas e Considerações

Limitações Reconhecidas

  • Cobertura: ~80% da base será processada (ignorando PDFs escaneados que requerem OCR)
  • Idiomas: Múltiplos idiomas podem apresentar variações na identificação de padrões de citação
  • Citações implícitas: A Fase 1 focará em citações explícitas; análise de influência doutrinária implícita ficará para fase futura
  • Qualidade dos PDFs: Possíveis erros de extração em documentos mal formatados
  • Ambiguidades: Casos com nomes similares podem requerer validação manual

Considerações Acadêmicas

  • Todas as limitações serão documentadas explicitamente na tese
  • Metodologia será reprodutível e auditável
  • Código será versionado e comentado
  • Decisões metodológicas serão justificadas com base em literatura pertinente
  • Testes de robustez serão realizados sempre que possível
